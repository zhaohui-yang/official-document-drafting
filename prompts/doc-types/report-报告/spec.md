# 报告规范

## 写作规则

- 适用场景：向上级汇报工作、反映情况、回复询问。
- 结构重点：主送单位、基本情况、工作进展、存在问题、下一步打算、落款、附注。
- 要素分层：`必备` 为主送单位、基本情况、工作开展情况、落款；`常见` 为工作成效、存在问题、下一步建议或下一步打算；`条件项` 为原因分析、需说明事项、附件；`项目自定义` 为附注联系人，本项目默认保留，但不把它视为所有报告通行必备项。
- 写作提醒：核心是“报告情况”，不要把请批事项混进正文主线。
- 分段职责：`基本情况` 写报告缘由、背景范围、总体态势和需要说明的起点信息；`工作开展情况` 写已经做了什么、采取了哪些措施、形成了什么阶段结果；`存在问题` 写当前短板、风险和困难，不回避矛盾；`下一步建议` 或 `下一步打算` 写准备如何推进、需要上级关注什么，但不改写成请批事项。
- 常见类型：部委和省级样例中的报告，至少可区分为 `上行工作报告`、`专项情况报告`、`答复询问报告`、`会议审议报告`；起草前应先判断属于哪一类，再决定结尾、称谓和板块取舍。
- 行文边界：上行报告一般只写一个主送机关，重在汇报情况、反映问题、答复询问，不以报告名义夹带请示批准事项；确需审批的，应改判为请示或请示、报告分写。
- 需要展示统计趋势、对比结果、现场情况、整改前后变化时，可在附件中加入 `图表`、`照片`、`示意图` 或 `截图说明`；正文重点仍是报告情况和结论，不以图代文。
- 结尾用语应与场景匹配：向上级报送工作和情况时，可用“特此报告。”；向人大、常委会、专题会议提交审议或审查的报告，宜用“以上报告，请审议”“以上报告，请予审查”等场景化结尾。
- 二级标题下优先采用“概述 + 一是、二是、三是”的展开方式：先用 3 至 5 句话概述本节总体情况，再分点展开，每一点一般写 5 至 7 句话，写清做法、进展、问题、成效或建议，不要只写口号。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 如用户已提供主送单位、发文单位或日期，则据实写入；未提供时保留 `[主送单位]`、`[发文单位]`、`[日期]` 等占位符，不省略对应结构。
- 本项目默认对报告保留附注联系人信息；如用户未提供，则保留联系人、固定电话、手机占位符，用户明确要求删除时再去掉。
- 常用结尾：特此报告。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，常见格式为“关于[事项]的报告”。
- 主送单位放在标题下空一行顶格书写，末尾用全角冒号；上行文一般只写一个主送机关更稳妥。
- 正文用 3 号仿宋体，首行缩进 2 字符；“基本情况、工作进展、存在问题、下一步打算”宜分层表达。
- 报告重在汇报，不应把请示事项混写进正文；如必须提出请求，应改判为请示或请示报告分写。
- 答复询问类报告宜围绕所问事项逐条回应，不机械套用“基本情况、存在问题、下一步打算”全套骨架。
- 正文结尾常用“特此报告。”，落款和日期置于文末右侧。

## 模板

```markdown
# 报告模板

## 标题

关于[事项名称]的报告

## 主送单位

[主送单位]：

## 一、基本情况

[交代事项背景、总体情况和报告缘由。]

## 二、工作开展情况

[分点写明已完成工作、主要做法和阶段成效。]

## 三、存在问题

[如实说明当前问题、风险或困难。]

## 四、下一步建议

[说明下一步工作安排或需要关注的事项。]

特此报告。

## 落款

[发文单位]  
[日期]

## 附注

（联系人：[联系人] 联系电话：[固定电话]，[手机号]）

## 附件（可选）

附件1：[图表或照片标题]

图1：[图片标题]
[图片描述]
说明：[说明文字]
```
