# 报告规范

## 写作规则

- 适用场景：向上级汇报工作、反映情况、回复询问。
- 结构重点：主送单位、基本情况、工作进展、存在问题、下一步打算、落款。
- 写作提醒：核心是“报告情况”，不要把请批事项混进正文主线。
- 二级标题下优先采用“概述 + 一是、二是、三是”的展开方式：先用 3 至 5 句话概述本节总体情况，再分点展开，每一点一般写 5 至 7 句话，写清做法、进展、问题、成效或建议，不要只写口号。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 如用户已提供主送单位、发文单位或日期，则据实写入；未提供时保留 `[主送单位]`、`[发文单位]`、`[日期]` 等占位符，不省略对应结构。
- 如属于需要报送联系人和联系电话的上行文，可在附注处填写；用户未提供时保留占位符。
- 常用结尾：特此报告。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，常见格式为“关于[事项]的报告”。
- 主送单位放在标题下空一行顶格书写，末尾用全角冒号；上行文一般只写一个主送机关更稳妥。
- 正文用 3 号仿宋体，首行缩进 2 字符；“基本情况、工作进展、存在问题、下一步打算”宜分层表达。
- 报告重在汇报，不应把请示事项混写进正文；如必须提出请求，应改判为请示或请示报告分写。
- 正文结尾常用“特此报告。”，落款和日期置于文末右侧。

## 模板

```markdown
# 报告模板

## 标题

关于[事项名称]的报告

## 主送单位

[主送单位]：

## 基本情况

[交代事项背景、总体情况和报告缘由。]

## 工作开展情况

[分点写明已完成工作、主要做法和阶段成效。]

## 存在问题

[如实说明当前问题、风险或困难。]

## 下一步建议

[说明下一步工作安排或需要关注的事项。]

特此报告。

## 落款

[发文单位]  
[日期]
```
