# 函规范

## 写作规则

- 适用场景：平行机关或不相隶属机关之间商洽、询问、答复事项。
- 结构重点：来往对象、事项依据、函告内容或回复结论、落款、附注。
- 要素分层：`必备` 为标题、主送单位、事项缘由、主要意见、落款；`常见` 为配合事项、来函或发函背景、具体协商事项；`条件项` 为反馈时限、附件；`项目自定义` 为附注联系人，本项目默认保留，但不等同函的通行必备要素。
- 写作提醒：平行沟通语气要得体，事项和依据要写清。
- 分段职责：`事项缘由` 写为什么来函、基于什么背景或来文；`主要意见` 写请求事项、协商意见或答复结论；`配合事项` 写需对方反馈、协办、补充材料或在何时前办结的事项，做到一函一事、事项闭合。
- 常见类型：部委和省级样例中的函，至少可区分为 `商洽函`、`询问函`、`请批函`、`答复函`；不同类型在结尾用语和语气强度上应有所区分。
- 行文语气：函属于平行或不相隶属机关之间的正式往来，措辞应平等、审慎、直接，不用命令式部署口吻代替商洽、询问或回复。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 如用户已提供主送单位、发函单位或日期，则据实写入；未提供时保留 `[主送单位]`、`[发函单位]`、`[日期]` 等占位符，不省略对应结构。
- 本项目默认对函保留附注联系人信息；如用户未提供，则保留联系人、固定电话、手机占位符，用户明确要求删除时再去掉。
- 常用结尾：特此函复。或 特此函达。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，宜写成“关于[事项]的函”。
- 主送单位通常放在标题下空一行顶格书写，末尾用全角冒号。
- 正文用 3 号仿宋体，首行缩进 2 字符；函文通常篇幅较短，事项、意见、请求或答复应直截了当。
- 如需分点，可按常规层级标题处理；篇幅短时也可不分层，保持一事一函的紧凑结构。
- 商洽、询问类函宜把反馈事项、反馈方式和反馈时限单独成句写清；答复类函则应优先把答复结论置前，不让对方通篇寻找结论。
- 结尾常用“特此函达”“特此函复”，落款和日期置于文末右侧。

## 模板

```markdown
# 函模板

## 标题

关于[事项名称]的函

## 主送单位

[主送单位]：

## 事项缘由

[说明来函或发函背景、目的和需要协商或答复的事项。]

## 主要意见

[写明商洽意见、请求事项或答复内容。]

## 配合事项

[如需对方反馈、办理或协同，写明具体要求和时限。]

## 落款

[发函单位]  
[日期]

## 附注

（联系人：[联系人] 联系电话：[固定电话]，[手机号]）
```
