# 通告规范

## 写作规则

- 适用场景：在一定范围内公布应当遵守或者周知的事项。
- 结构重点：发布缘由、通告事项、执行要求、落款、附注。
- 要素分层：`必备` 为发布缘由、通告事项、执行要求、落款；`常见` 为适用区域或对象、行为要求分条、执行时间；`条件项` 为处罚依据、咨询方式、附件；`项目自定义` 为附注联系人，本项目默认保留，但不视为通告的通行必备项。
- 写作提醒：强调“周知”和“遵守”，适用范围通常比公告更具体。
- 分段职责：`发布缘由` 写为什么要通告、适用区域和目的；`通告事项` 写群众或特定对象应知应守的具体内容，可按时间、区域、对象、行为要求分条写；`执行要求` 写执行时间、责任单位、处罚依据、咨询电话或注意事项。
- 核心要素：通告通常要把适用区域、适用对象、起止时间、应遵守或禁止的行为、处理依据和咨询渠道写清，尤其不能遗漏时间范围和空间范围。
- 常见起句：部委和省级样例中的通告，正文常直接使用“现通告如下”或相近句式，总起后分条列明事项。
- 需要面向公众说明区域范围、交通管制、封闭路线、办理路径时，可配发 `附图`、`位置示意图`、`流程图` 等附件；正文只作提要，图件说明应写清适用时间、区域和使用边界。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 本项目默认对通告保留附注联系人信息；如用户未提供，则保留联系人、固定电话、手机占位符，用户明确要求删除时再去掉。
- 常用结尾：特此通告。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，宜写明通告事项和适用范围。
- 通告一般不设主送单位，正文可按事项、区域、时限、要求分条列明。
- 正文用 3 号仿宋体，首行缩进 2 字符；面对社会公众时，条目表达要比内部文件更直接清楚。
- 处罚依据、执行时间、咨询电话、责任单位等信息宜单独成条，不要埋在叙述段中。
- 发布单位和日期通常置于文末右侧；公告性文本最终仍应使用本单位正式发布模板。

## 模板

```markdown
# 通告模板

## 标题

关于[事项名称]的通告

## 发布缘由

[说明通告发布的背景、原因或目的。]

## 通告事项

一、[事项一]  
[写明周知或应当遵守的具体事项。]

二、[事项二]  
[补充要求。]

## 执行要求

[写明适用范围、执行时间或责任要求。]

## 落款

[发文单位]  
[日期]

## 附注

（联系人：[联系人] 联系电话：[固定电话]，[手机号]）

## 附件（可选）

附件1：[附图标题]

图1：[图片标题]
[图片描述]
说明：[说明文字]
注：[适用时间、区域或使用边界]
```
