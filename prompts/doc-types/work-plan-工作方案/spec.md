# 工作方案规范

## 写作规则

- 适用场景：专项行动、阶段性工作、制度落地、项目推进。
- 结构重点：制定背景、总体要求、工作目标、主要任务、实施步骤、保障措施。
- 要素分层：`必备` 为标题、制定背景、总体要求、工作目标、主要任务、实施步骤、保障措施；`常见` 为适用范围、任务分工、时间安排；`条件项` 为责任分工、附件。
- 写作提醒：任务要拆清、步骤要可执行、保障措施要能落地。
- 分段职责：`制定背景` 写为什么要制定方案、依据和现实需求；`总体要求` 写指导思想、基本原则和总体安排；`工作目标` 写预期成效、量化指标或时间要求；`主要任务` 写具体工作、责任分工和完成标准；`实施步骤` 写阶段安排和时间节点；`保障措施` 写组织领导、资源保障、督导考核和纪律要求。
- 与通知配套关系：部委和省级样例中的工作方案、实施方案，较长者往往作为通知附件印发；如果方案本身是附件，正文无需再重复通知式部署语气。
- 任务承载方式：责任清单、任务台账、时间安排表、路线图、责任分工表等信息较多时，可通过附件或附表承载，正文重点写框架、原则和关键任务，不把表格内容硬写成大段文字。
- 如需呈现流程链条、施工路线、点位分布、系统操作路径、时间路线图等，更适合在附件中加入 `流程图`、`示意图`、`路线图`、`时间轴图` 或 `截图说明`；只有明显属于技术手册或操作指南时，才进一步使用 `附录`。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：可不单独设置套话式结尾，通常以措施或附件收束。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常居中排布，可用 2 号小标宋体，常见格式为“[事项]工作方案”或“[事项]实施方案”。
- 一般不设主送单位；如作为通知附件或专项文件，可在标题下补充适用范围或执行期限说明。
- 正文通常用 3 号仿宋体，首行缩进 2 字符；“总体要求、目标任务、实施步骤、保障措施”宜分层写清。
- 工作方案常含附件、流程表、责任分工等内容，正文中的时间节点和责任主体应单独成句或成条。
- 若方案要直接用于发文或会议印发，最终版式仍应与本单位成套模板保持一致。

## 模板

```markdown
# 工作方案模板

## 标题

关于开展[事项名称]的工作方案

## 一、制定背景

[填写制定背景、政策依据或现实需求。]

## 二、总体要求

[填写指导思想、基本原则或总体安排。]

## 三、工作目标

[填写目标任务、预期成效或时间要求。]

## 四、主要任务

（一）[任务一]

[填写具体工作内容、责任分工和完成标准。]

（二）[任务二]

[填写具体工作内容、责任分工和完成标准。]

## 五、实施步骤

[填写阶段安排、时间节点和推进节奏。]

## 六、保障措施

[填写组织领导、督导检查、资源保障、纪律要求等。]

## 附件（可选）

附件1：[流程图或示意图标题]

图1：[图片标题]
[图片描述]
说明：[说明文字]
```
