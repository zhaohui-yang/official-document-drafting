# 讲话稿规范

## 写作规则

- 适用场景：领导讲话、会议发言、动员部署、总结点评。
- 结构重点：开场说明、形势判断、重点任务、工作要求、结束语。
- 要素分层：`必备` 为标题、开场说明、重点任务、结束语；`常见` 为称谓、形势判断、工作要求、重点任务分层；`条件项` 为附件或提纲。
- 写作提醒：节奏可以更强，但仍要保持机关文风和可执行表达。
- 分段职责：`开场说明` 写会议场合、讲话目的和总体判断；`形势判断` 写当前背景、问题和意义；`重点任务` 写接下来要抓的重点工作和举措；`工作要求` 写组织领导、责任压实、纪律作风和落实要求；`结束语` 简洁收束到号召、感谢或明确要求。
- 类型判断：讲话稿要先区分 `讲话`、`发言`、`致辞`、`主持词` 等具体场景；同样是会场材料，动员讲话、总结讲话、致辞和主持串词的语气、称谓和长度差异明显，不宜套一个模板。
- 口头表达边界：讲话稿用于现场宣读时，宜多短句、少嵌套，称谓通常只在开头出现一次；文件号、长书面引文和过长定语从句应压缩，避免影响口头表达。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：谢谢大家。或 让我们共同抓好落实。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常居中排布，可用 2 号小标宋体，也可按会场材料习惯使用黑体标题。
- 讲话稿一般不设主送单位，开头如有称谓，可顶格单独成段并加全角冒号。
- 正文通常用 3 号仿宋体，首行缩进 2 字符；“一、二、三”式层级结构较常见，重点句可适度单列。
- 讲话稿更强调口头表达的节奏，可使用较短段落和提示性过渡句，但仍应保持正式文字风格。
- 如需会场宣讲稿、提词稿或分发稿，应在定稿后按实际使用场景再做版式压缩或放大。

## 模板

```markdown
# 讲话稿模板

## 标题

在[会议或活动名称]上的讲话

## 称谓

[各位领导、同志们：]

## 开场说明

[填写会议背景、讲话目的或总体评价。]

## 形势判断

[填写当前形势、问题分析或背景引入。]

## 重点任务

一、[任务一]

[填写重点工作。]

二、[任务二]

[填写重点工作。]

## 工作要求

[填写组织要求、纪律要求或落实要求。]

## 结束语

[填写总结性收束语。]
```
