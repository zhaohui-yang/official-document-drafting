# 议案规范

## 写作规则

- 适用场景：具有法定主体和程序要求的议案事项。
- 结构重点：案由、依据、主要内容、审议请求。
- 写作提醒：不要泛化使用，主体和程序不符合时应改用其他文种。
- 分段职责：`提请审议事项` 要开门见山写清提交审议什么；`依据与理由` 写法定依据、现实必要性和可行性；`议案内容` 写拟议条款、方案安排或处理建议；结尾的审议请求要明确落到“请予审议”或相近表述。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：请予审议。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，宜写明提请审议事项。
- 议案一般不设普通主送单位，正文常先写提请依据和必要性，再写议案内容。
- 正文用 3 号仿宋体，首行缩进 2 字符；如涉及多个审议点，宜逐项列明，便于审查。
- 法定程序性内容、附件清单、提案人署名等应按具体会议制度或法定机关要求处理。
- 形成正式议案文本前，应以对应法定会议系统模板进行最终复核。

## 模板

```markdown
# 议案模板

## 标题

关于[事项名称]的议案

## 提请审议事项

[明确提请审议的事项内容。]

## 依据与理由

[说明提出议案的法律政策依据、现实背景和必要性。]

## 议案内容

[写明具体议案条款、方案或建议内容。]

## 落款

[提请单位]  
[日期]
```
