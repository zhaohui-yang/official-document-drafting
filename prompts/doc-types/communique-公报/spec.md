# 公报规范

## 写作规则

- 适用场景：公开发布重要决定、重大事项或重要会议情况。
- 结构重点：背景或会议概况、主要内容、结果性信息、结语。
- 要素分层：`必备` 为背景概况、主要内容；`常见` 为会议名称或届次、审议或通过情况、主要决定、重要成果；`条件项` 为后续安排或发布说明。
- 写作提醒：面向社会公开，强调权威和结果，不写成内部简报。
- 分段职责：`背景概况` 写会议召开、审议程序、重大事项背景和权威来源；`主要内容` 写正式公布的决定、成果、原则和重要信息；`结语` 写后续安排、生效说明或公开发布口径，不延展内部过程性细节。
- 常见结构顺序：部委和省级公开样例中的公报，常按“会议召开情况或发布背景 -> 审议通过、决定、发布的主要事项 -> 对外说明或号召性表述”组织，不机械套用内部材料的“背景、问题、措施”写法。
- 行文边界：公报重在公开权威结果和正式信息，通常不设主送单位，也不保留一般事务性附注联系人；结尾可自然收束，不必机械套用“特此公报。”这类固定套话。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：通常以结果性信息、说明性语句或号召性表述自然收束，不强制固定套话。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，可包含会议名称、时间范围或发布主体。
- 公报一般不设主送单位，正文多为公开发布内容，段落可按议题、成果或事项分层展开。
- 正文用 3 号仿宋体，首行缩进 2 字符；重要结论、决定事项可单独成段或分条。
- 如含会议概况、主要精神、后续安排等模块，建议分别设一级标题，保持公开文本的清晰性。
- 是否保留署名、日期、发布说明，应以实际发布机制和既有公报模板为准。

## 模板

```markdown
# 公报模板

## 标题

[会议名称/事项名称]公报

## 背景概况

[说明会议召开情况、审议过程或重大事项背景。]

## 主要内容

一、[主要事项一]  
[写明正式公布的结果、决定或权威信息。]

二、[主要事项二]  
[补充内容。]

## 结语

[如需，可写后续安排或公开说明。]
```
