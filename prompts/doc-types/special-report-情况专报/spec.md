# 情况专报规范

## 写作规则

- 适用场景：向领导或上级报送专题信息、风险情况、舆情态势、突发事件进展。
- 结构重点：事项概况、最新进展、风险研判、工作建议。
- 要素分层：`必备` 为标题、事项概况、最新进展、风险研判、工作建议；`常见` 为报送对象、时间点说明、已采取措施；`条件项` 为影响评估、附件。
- 写作提醒：先列事实，再写判断；时间点、来源和待核实事项必须清楚。
- 分段职责：`事项概况` 写事件起因、时间地点、涉及主体和当前基本态势；`最新进展` 只写已核实的新情况、已采取措施和直接结果；`风险研判` 写趋势判断、主要风险点和可能影响，并与已核实事实分开；`工作建议` 写建议采取的处置措施、报送建议或领导需关注事项。
- 时效标识：时效性强的专报，宜在标题下说明报送时间、信息截止时点、信息来源范围或核实口径，避免不同批次专报之间时间口径混乱。
- 表述边界：专报中的“事实”“研判”“建议”应严格分栏或分段；未经核实的信息可以列为“待核实”“网传情况”“初步掌握”，不得与已核实情况混写。
- 对传播链路、舆情走势、现场态势、区域分布等需要直观看图的内容，可在附件中加入 `截图`、`趋势图`、`关系图`、`地图示意图` 等图片说明；正文不直接插图，以免破坏专报的快速阅知结构。
- 对“最新进展”“风险研判”“工作建议”等综合性板块，二级标题下优先先用 3 至 5 句话概述，再用 `一是、二是、三是` 分点展开，每一点一般写 5 至 7 句话；确属单条进展、单项风险提示或单项建议时可从简，但不得空泛。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：以上情况专报，请审阅。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常居中排布，可用 2 号小标宋体，宜直接点明专题、风险点或事件主题。
- 一般不设主送单位；如为定向报送领导，可在标题下以说明行标明报送对象和日期。
- 正文通常用 3 号仿宋体，首行缩进 2 字符；“基本情况、分析研判、风险提示、工作建议”宜分层写清。
- 专报要强调时效性和判断力，关键时间、数据、趋势和建议可单独成段突出，不宜铺陈过长背景。
- 如涉及敏感信息、涉密分级或内部流转标记，应以本单位专报模板和保密要求为准。

## 模板

```markdown
# 情况专报模板

## 标题

[事项名称]情况专报

## 报送对象

[报送对象]

## 事项概况

[填写事件背景、涉及主体、时间地点和当前基本态势。]

## 最新进展

[填写已核实的最新动态、已采取措施和当前结果。]

## 风险研判

[填写主要风险点、趋势判断和待关注问题；判断应与事实区分开。]

## 工作建议

[填写建议采取的措施、报送建议或处置建议。]

## 附件（可选）

附件1：[图片或图表示意]

图1：[图片标题]
[图片描述]
说明：[说明文字]
注：[必要提示]
```
