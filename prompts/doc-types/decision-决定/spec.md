# 决定规范

## 写作规则

- 适用场景：对重要事项作出部署、奖惩、处理、调整。
- 结构重点：作出决定的背景和依据、决定事项、执行要求、落款。
- 要素分层：`必备` 为背景依据、决定事项、执行要求、落款；`常见` 为适用范围、调整依据、决定事项分条、责任主体；`条件项` 为执行时限、附件。
- 写作提醒：结论要明确，力度要强，不能写成情况汇报。
- 分段职责：`背景依据` 写为什么要作出决定、依据是什么；`决定事项` 写决定了什么、调整了什么、奖惩了什么；`执行要求` 写谁负责落实、按什么标准落实、什么时间落实到位，不能把决定事项稀释成一般倡议。
- 常见类型：部委和省级样例中的决定，至少可区分为 `部署性决定` 与 `奖惩、撤销、调整、公布类决定`；部署性决定可保留背景依据和执行要求，奖惩、撤销、调整类决定则宜更直接地进入决定事项和生效安排。
- 事项表达：决定事项应写成已经作出的处理结论、制度安排或调整结果，不写成讨论中的倾向性意见；涉及撤销、废止、奖惩、任免、调整的，应把对象、事项和效力边界写清。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：现作出如下决定。请认真贯彻执行。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，标题宜直接写明决定事项。
- 决定类文本一般不单列主送单位，正文通常先写背景依据，再写决定事项和执行要求。
- 正文用 3 号仿宋体，首行缩进 2 字符；“决定事项”部分宜分条列出，便于执行。
- 一级标题通常用黑体，奖惩、处理、部署等关键结论应单独成层，不宜埋在叙述段中。
- 落款与日期通常置于文末右侧；如属于会议通过的决定，可按会议文件格式处理。

## 模板

```markdown
# 决定模板

## 标题

关于[事项名称]的决定

## 背景依据

[说明作出决定的背景、依据和必要性。]

## 决定事项

一、[决定事项一]  
[写明处理、部署、奖惩、调整等核心决定内容。]

二、[决定事项二]  
[补充具体要求。]

## 执行要求

[明确执行范围、责任主体和落实要求。]

## 落款

[发文单位]  
[日期]
```
