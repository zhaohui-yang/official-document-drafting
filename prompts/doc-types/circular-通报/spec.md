# 通报规范

## 写作规则

- 适用场景：表彰先进、批评错误、传达重要情况。
- 结构重点：基本情况、评价或问题分析、有关要求、落款。
- 要素分层：`必备` 为基本情况、评价分析、工作要求、落款；`常见` 为处理结果、表彰批评情况、原因分析；`条件项` 为主送范围、整改要求、附件。
- 写作提醒：事实、评价、要求三层要分明，避免只写态度不写事实。
- 分段职责：`基本情况` 写清事情经过、涉及对象、结果和影响；`评价分析` 写成绩经验、问题性质、原因教训或警示意义；`工作要求` 写整改、追责、学习推广或防范措施，不能把评价分析写成口号。
- 常见类型：部委和省级样例中的通报，至少可区分为 `表彰性通报`、`批评性通报`、`情况通报`；表彰性通报重在经验和推广要求，批评性通报重在事实、性质、责任和整改，情况通报重在通报重要情况和提出工作要求。
- 事实先行：批评性、情况性通报应先把事实、过程、结果和影响写清，再写性质判断和要求；没有查清的责任、数据和结论，不提前写成定论。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：特此通报。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，宜直接点明被通报事项或对象。
- 通报可根据需要设置主送范围；如设置主送单位，标题下空一行顶格书写。
- 正文用 3 号仿宋体，首行缩进 2 字符；事实、评价、要求宜分层表达，避免混在一个长段中。
- 表彰类、批评类和情况通报都应保证层次清楚，一级标题可用黑体突出重点。
- 落款和日期通常置于文末右侧；对外发布场景仍以正式发文模板为准。

## 模板

```markdown
# 通报模板

## 标题

关于[事项名称]的通报

## 一、基本情况

[说明事件、工作、典型案例或问题的基本情况。]

## 二、评价分析

[写明经验做法、问题性质、原因分析或警示意义。]

## 三、工作要求

[明确吸取教训、抓好整改、学习推广或后续措施。]

## 落款

[发文单位]  
[日期]
```
