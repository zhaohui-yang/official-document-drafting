# 汇报材料规范

## 写作规则

- 适用场景：向领导汇报、向检查组报送、向上级报备。
- 结构重点：工作进展、主要成效、存在问题、下一步措施。
- 要素分层：`必备` 为工作进展、主要成效、存在问题、下一步措施；`常见` 为基本情况、主要做法、需领导关注事项；`条件项` 为附件、落款。
- 写作提醒：重在让阅读者快速把握情况、问题和需要关注的事项。
- 分段职责：`工作进展` 写推进了什么、推进到哪一步、采取了哪些做法；`主要成效` 写形成了什么结果、亮点和经验；`存在问题` 写当前短板、堵点和风险；`下一步措施` 写下一步准备怎么干、需协调什么事项；`需要说明事项` 只写需领导关注、拍板或知悉的补充情况。
- 常见起句：向领导、检查组或上级汇报的材料，正文开头常直接使用“现将有关情况汇报如下”或相近句式，引出主体内容。
- 事项聚焦：汇报材料除交代情况外，还应把需要领导关注、协调、拍板或批示的事项单独拎出来，不把关键决策点埋在一般工作进展中。
- 汇报材料较适合在附件中增设 `现场照片`、`数据图表`、`流程图`、`对比图`、`架构图` 等图片说明，作为口头汇报或后续转 PPT 的辅助材料；正文仍以文字判断和工作结论为主。
- 综合性板块优先采用“概述 + 一是、二是、三是”的展开方式：二级标题下先用 3 至 5 句话概述，再分点展开，每一点一般写 5 至 7 句话，确保情况、问题、成效和建议都有实质内容。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：有关情况汇报如下。或 以上汇报，请审示。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常居中排布，可用 2 号小标宋体，也可按汇报场景采用更简洁的黑体标题。
- 一般不设主送单位；如明确报送对象，可在标题下以副题或说明行体现。
- 正文通常用 3 号仿宋体，首行缩进 2 字符；“基本情况、工作进展、问题不足、下一步打算”宜分层呈现。
- 面向领导阅示的材料应多短段落、少长句，关键数据、问题和建议可单独成段突出。
- 若后续要转 PPT、讲话提纲或会议汇报稿，建议先保持结构清楚，再做场景化排版调整。

## 模板

```markdown
# 汇报材料模板

## 标题

关于[事项名称]的汇报材料

## 一、工作进展

[填写工作推进情况、阶段安排和主要节点。]

## 二、主要成效

[填写取得的成效、典型经验和阶段成果。]

## 三、存在问题

[填写当前短板不足、堵点难点和风险问题。]

## 四、下一步措施

[填写下一步工作安排、改进措施和需要协调事项。]

## 五、需要说明事项（可选）

[填写需补充说明或请领导关注的事项。]

## 落款（可选）

[汇报单位]
[日期]

## 附件（可选）

附件1：[图片或图表标题]

图1：[图片标题]
[图片描述]
说明：[说明文字]
```
