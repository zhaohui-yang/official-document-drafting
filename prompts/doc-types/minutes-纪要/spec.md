# 纪要规范

## 写作规则

- 适用场景：记载会议主要情况和议定事项。
- 结构重点：会议基本信息、出席、参加、记录、正文、责任分工、后续要求、版记；如信息完整或用户未限制篇幅，可从严保留 `请假（可选）`、`附件（可选）` 等扩展结构，后续由用户删减。
- 要素分层：`必备` 为会议基本信息、正文；`常见` 为出席、参加、记录、责任分工、后续要求；`条件项` 为请假、附件、按议题分层写法；`地方或系统样式` 为期号、分送、发放范围等尾部信息；`项目自定义` 为版记中的主送、抄送、审核，本项目默认保留，但不视为所有纪要通行必备项。
- 写作提醒：只写形成共识的内容，责任单位和时限尽量明确；默认按“宁可稍全、后删减”的思路保留结构和约束，不主动把纪要压缩成过短的简讯式文字。
- 分段职责：`会议基本信息` 写会议名称、时间地点、主持和议题；`出席` 写正式出席会议人员；`参加` 写参加或列席会议人员；`记录` 写记录人；`正文` 先写导语，再用“会议认为，……”写会议形成的总体判断、一致意见和基本认识，用“会议决定，……”写正式确定的事项、结论和处置意见，两者均作为正文起句连续书写，不另设同名小标题；`责任分工` 写谁在何时前完成什么任务；`后续要求` 写督办、反馈、保密和跟踪落实要求；`版记` 默认保留主送、抄送、审核。
- 对正文中的“会议认为，……”“会议决定，……”等综合性内容，优先先用 3 至 5 句话概述，再用 `一是、二是、三是` 分点展开，每一点一般写 5 至 7 句话；对责任分工、完成时限、单项决定等执行性条目可分条列写，但不能空泛。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：一般不写寒暄式结尾，直接收束到责任和要求。
- 总体质量要求：纪要应围绕会议主题提炼形成书面结论，措辞宜严谨、准确、精炼、便于执行和操作，且应与会议形成的决定、共识和口径保持一致，不得出现理解歧义。
- 记录边界：纪要不是逐字会议记录，不机械照搬个人原话，不罗列无结论的来回讨论，不铺陈会场过程性细节；除确有必要，不单独写个人姓名后的发言摘录。
- 共识边界：纪要以记载会议形成的共识、决定、要求为主；未经会议形成一致意见的事项，不写成既定结论，可写“继续研究”“补充完善后另报”“另行专题研究”。
- 议题顺序：多议题会议宜按照会议实际议题顺序组织正文，一事一段、一事一条或一事一组提示语，不同议题不要交叉混写。
- 单项与多项写法：单一专题会议可采用“导语 + 会议认为，…… + 会议决定，…… + 责任分工 + 后续要求”的集中写法；多议题会议可采用“导语 + 按议题逐项写听取/研究/审议情况 + 对应决定事项”的分项写法。
- 出席信息写法：`出席` 一般写正式出席会议并参与议定事项的人员；`参加` 一般写参加或列席会议人员；如材料中存在缺席情况，可增设 `请假（可选）`，写明请假人员或单位。
- 名单口径：纪要中的人员、单位和职务写法应前后一致，避免同一单位在不同段落中简称、全称混用；同一名单内的排序宜按职务、层级或既定顺序统一排列。
- 开头内容边界：导语除交代时间、地点、主持、议题外，宜同步写明“会议听取了什么、研究了什么、审议了什么”；不要用大段背景材料替代会议导语。
- 背景控制：如需补充背景，只写与本次会议讨论直接相关的背景、依据和现状，不把纪要写成长篇汇报材料或宣传报道。
- 语言风格：多用“研究、审议、明确、同意、要求、决定、原则同意、暂缓、退回修改、抓紧落实”等动作词；少用口语化词语、评论性夸张表述和空泛口号。
- 执行闭环：纪要中凡涉及任务交办、方案调整、协调事项、问题整改、材料报送、公开发布、进一步论证等内容，原则上都应对应责任单位、时限、成果形态或反馈要求。
- 督办反馈：后续要求中宜写明跟踪督办、办理反馈、请示报告、保密管理、材料归档、对外口径等安排，避免会议决定形成后无落实抓手。
- 保密边界：涉及未公开事项、敏感事项或内部研判的纪要，宜在后续要求中写明不得擅自扩散、对外发布和引用；如用户未要求，可不单设保密标题，但应在措辞上保守。
- 附件边界：会议审议材料、清单、方案、名单、汇报提纲、统计表等如需随纪要转送，可增设 `附件（可选）`；未附附件时，不虚构附件名称和数量。
- 审定口径：纪要作为结果性正式文本，原则上应体现“以最终审定稿为准”的口径；项目模板中的 `审核` 属内部模板字段，可保留，但不把内部流转说明写进正文。
- 版记边界：部委、省级和地方系统样例中，纪要尾部可见 `分送`、`抄送`、`发放范围` 等系统样式；本项目默认保留 `主送、抄送、审核`，属于内部模板定制，不上升为纪要通行必备格式。
- 常见类型与写法差异：
  - `常务会议纪要`：适用于政府、党组、党委等常务会议，标题常见为“第[次数]次常务会议纪要”或“[单位名称]第[次数]次常务会议纪要”；开头宜先写会议时间、地点、主持和议题，再写听取、研究、审议了哪些事项；正文重在“会议认为，……”“会议强调，……”和“会议决定，……”等提示语起句后的连续表述，语言要稳、硬、概括性强。
  - `办公会议纪要`：适用于机关、部门、单位内部办公会议，标题常见为“办公会议纪要”或“[单位名称][日期]办公会议纪要”；开头除会议基本信息外，宜直接写会议围绕哪些具体工作进行了研究；正文重在协调事项、任务安排和责任落实，语言可比常务会议纪要更直接、更执行化。
  - `专题会议纪要`：适用于围绕某一项重点工作、项目、问题、活动召开的专题研究会，标题常见为“关于[事项名称]专题会议纪要”或“[事项名称]专题会议纪要”；开头要先点明专题背景和会议研究范围；正文重在问题分析、研究意见、会议决定和后续推进安排，宜突出事项单一、结论集中。
  - `联席/协调/会商会议纪要`：适用于多部门、多单位共同研究协同事项，标题常见为“[事项名称]联席会议纪要”“[事项名称]协调会议纪要”或“[事项名称]会商会议纪要”；开头宜先写参会单位和会商事项；正文重在各方共识、职责边界、配合机制和时间节点，责任分工要写到具体单位。
  - `座谈/研讨会议纪要`：适用于调研座谈、征求意见、专题研讨等场景，标题常见为“[事项名称]座谈会纪要”或“[事项名称]研讨会议纪要”；开头宜先写会议背景、参加范围和讨论主题；正文重在归纳意见、提炼共识、明确后续研究方向，不宜误写成已经形成刚性决定的执行纪要。
- 起草纪要前，应先判断属于哪一种类型，再决定标题写法、开头句式和正文重心；不能把座谈会纪要写成常务会议纪要，也不能把专题协调纪要写成纯记录式流水账。
- 标题写法宜单一明确：通常采用“第[次数]次常务会议纪要”“[事项名称]专题会议纪要”“[事项名称]联席会议纪要”“[事项名称]座谈会纪要”等稳定格式，不在一个标题中并列堆放多个中心事项。
- 标题与内容应一致：标题写“专题会议纪要”，正文就应围绕单一专题集中展开；标题写“办公会议纪要”或“常务会议纪要”，正文就应允许多议题并行，但仍需按议题分层组织。
- 开头写法宜固定化：通常先写“何时、何地、谁主持，围绕什么事项召开会议”，再写“会议听取了、研究了、审议了什么”；不要一开头就直接进入“会议认为”，也不要先写大段背景抒情。
- 提示语宜按语义使用：`会议听取了` 用于听取汇报，`会议研究了` 用于讨论事项，`会议审议了` 用于需形成正式意见的事项，`会议认为，` 用于总体判断，`会议指出，` 用于问题、成绩、形势和意义，`会议强调，` 用于重申原则和重点要求，`会议要求，` 用于部署具体任务，`会议决定，` 用于写明形成的结论和责任分工。
- 提示语可组合但不能混乱：同一事项下可按“会议听取了……。会议认为，……。会议强调，……。会议决定，……”的顺序连续书写，但不要把提示语无序堆叠，也不要把本该放在“会议要求，……”中的执行任务提前写进“会议认为，……”。
- 常务和办公会议可按议题形成“一、关于[议题一]……”“二、关于[议题二]……”的组织方式，每个议题内部再使用对应提示语；专题会议一般不必再套太多层级，优先保持单线展开。
- `会议认为，……` 和 `会议决定，……` 默认作为正文起句使用，不另设“会议认为”“会议决定”同名二级标题；如内容较长，可在同一正文板块内连续成段书写，也不要把提示语单独抬成小标题。
- `会议决定，……` 后的内容必须可执行：应逐条编号或在段内依次写明“同意什么、明确什么、由谁负责什么、何时完成什么”；未形成一致结论的，不写成“会议决定”，可写“继续研究”“补充完善后再报”“另行专题研究”。
- 责任和时限应单独写清：可以单设 `责任分工`，也可以在“会议决定，……”后的各项中直接写责任单位和完成时限，但同一份纪要内应保持一种写法，不要一半单列、一半散落。
- 事项表达宜完整闭环：对审议通过、原则同意、修改完善后执行、暂缓实施、继续征求意见、另行上会等不同处理结果，应使用对应措辞，不把不同处理状态混写为一个模糊结论。
- 对外公开与内部使用口径要区分：如纪要将用于公开发布，正文宜进一步压缩内部流程、争议意见和敏感细节；如用于内部执行，可适当保留更细的责任、时限、流程和反馈要求。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，会议名称和“纪要”可合成完整标题。
- 本项目中的纪要模板默认在文首列出时间、地点、主持人、出席、参加、记录等信息，在文末版记中保留主送、抄送、审核。
- 正文用 3 号仿宋体，首行缩进 2 字符；“会议认为，……”和“会议决定，……”一般直接作为正文提示语连续书写，不另设同名小标题。
- “会议决定，……”后的事项建议逐条编号，责任单位、完成时限单独表达，便于执行和跟踪。
- 人员信息、会议信息等元数据宜集中排在正文前，不宜散插进正文各段；名单较长时可分段列写，但不宜破坏信息完整性。
- 多议题纪要的正文层级宜保持统一：同一份纪要中，要么按议题编号组织，要么按“会议认为/会议决定/责任分工”组织，不宜两种结构大幅交叉。
- 项目模板默认在文末保留 `主送、抄送、审核`，这是内部模板口径；如用户单位有其他版记格式，可按用户模板覆盖。
- 纪要正文后默认不单列落款和印发信息；如用户所在单位另有固定模板，再按其要求调整。

## 模板

```markdown
# 会议纪要模板

## 会议基本信息

- 会议名称：[会议名称]
- 时间：[时间]
- 地点：[地点]
- 主持人：[主持人]

## 出席

[出席人员]

## 参加

[参加人员]

## 请假（可选）

[请假人员]

## 记录

[记录人]

[导语段，交代会议围绕什么事项召开、听取了什么、研究了什么、审议了什么。]

会议认为，[概述会议对当前情况的判断。]

会议决定，一、[议定事项一]
[补充说明。]

二、[议定事项二]
[补充说明。]

## 责任分工

[责任单位]于[时间]前完成[任务]。

## 后续要求

[跟踪督办、报送反馈、保密要求等。]

## 附件（可选）

1. [附件名称]

## 版记

主送：[主送单位]

抄送：[抄送单位]

审核：[审核人]
```
