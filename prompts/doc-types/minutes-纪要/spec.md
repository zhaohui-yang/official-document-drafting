# 纪要规范

## 写作规则

- 适用场景：记载会议主要情况和议定事项。
- 结构重点：会议基本信息、会议认为、议定事项、责任分工、后续要求。
- 写作提醒：只写形成共识的内容，责任单位和时限尽量明确。
- 分段职责：`会议基本信息` 写会议名称、时间地点、主持、参会范围和议题；`会议认为` 写会议形成的总体判断、一致意见和基本认识，不写个人零散发言；`议定事项` 写正式确定的事项、结论和处置意见；`责任分工` 写谁在何时前完成什么任务；`后续要求` 写督办、反馈、保密和跟踪落实要求。
- 对“会议认为”“重点议题”“后续要求”等综合性内容，二级标题下优先先用 3 至 5 句话概述，再用 `一是、二是、三是` 分点展开，每一点一般写 5 至 7 句话；对责任分工、完成时限、单项决定等执行性条目可从简单列，但不能空泛。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：一般不写寒暄式结尾，直接收束到责任和要求。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，会议名称和“纪要”可合成完整标题。
- 纪要一般不设主送单位，标题后可依次列出会议时间、地点、主持人、参会人员等基本信息。
- 正文用 3 号仿宋体，首行缩进 2 字符；“会议认为”“议定事项”“责任分工”宜分层列明。
- 议定事项建议逐条编号，责任单位、完成时限单独表达，便于执行和跟踪。
- 纪要是否单列落款、日期或印发说明，应以本单位会议纪要模板为准。

## 模板

```markdown
# 会议纪要模板

## 会议基本信息

- 会议名称：[会议名称]
- 时间：[时间]
- 地点：[地点]
- 主持人：[主持人]
- 参会范围：[参会范围]

## 会议认为

[概述会议对当前情况的判断。]

## 议定事项

一、[议定事项一]  
[补充说明。]

二、[议定事项二]  
[补充说明。]

## 责任分工

[责任单位]于[时间]前完成[任务]。

## 后续要求

[跟踪督办、报送反馈、保密要求等。]
```
