# 批复规范

## 写作规则

- 适用场景：答复下级机关请示事项。
- 结构重点：对来文的简要回顾、批复结论、执行要求、落款。
- 要素分层：`必备` 为主送单位、来文依据、批复意见、落款；`常见` 为执行要求；`条件项` 为来文文号或时间、批复依据、执行时限、报备或反馈要求、附件。
- 写作提醒：必须直接回应原请示事项，结论不可含糊。
- 分段职责：`来文依据` 只写收到什么请示、围绕什么事项请示；`批复意见` 必须正面作出同意、不同意、原则同意或需调整完善的结论；`执行要求` 写如何落实、何时报送、还需补哪些程序，不能把批复结论藏在执行要求里。
- 常见起句：部委和省级样例中的批复，首段常用“你单位《关于……的请示》（文号）收悉。现批复如下：”或相近句式，先交代来文依据，再直接进入批复意见。
- 对应关系：批复通常实行“一请示一批复”；如原请示包含多个具体事项，批复意见应逐项对应作出明确答复，不笼统写“原则同意”后不交代边界。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：特此批复。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，宜写明被批复事项。
- 如设置主送单位，放在标题下空一行顶格书写；正文首段常直接写来文依据和“现批复如下”。
- 正文用 3 号仿宋体，首行缩进 2 字符；层级标题按黑体、楷体、仿宋加粗的常规层次处理。
- 批复意见宜按条列出，执行要求另起段或另设条目，不要把批复结论埋在长段中。
- 批复正文宜短、直、硬，不重写下级请示中的长篇背景；不同意、暂不批准或要求补充完善的，应把否定性结论和后续补正要求写清。
- 落款中的发文单位和日期置于文末右侧，正式批复仍应套用本单位发文模板复核。

## 模板

```markdown
# 批复模板

## 标题

关于[请示事项名称]的批复

## 来文依据

[写明收到的请示标题、文号和来文时间。]

## 批复意见

[明确同意、不同意、原则同意或需调整完善的具体意见。]

## 执行要求

[写明后续执行要求、报备要求或注意事项。]

此复。

## 落款

[批复单位]  
[日期]
```
