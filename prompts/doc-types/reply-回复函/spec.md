# 回复函规范

## 写作规则

- 适用场景：对来函、咨询、协调事项作出正式回复。
- 结构重点：来文回顾、答复意见、执行要求、落款、附注。
- 写作提醒：先回应对方关切，再写可执行结论，不绕弯。
- 分段职责：`来文回顾` 简要写收到什么来函、对方主要关切是什么；`答复意见` 逐项回应是否同意、如何处理、需要补什么材料；`执行要求` 写后续对接、办理时限和责任分工，不要把答复结论拖到最后一句才出现。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 本项目默认对回复函保留附注联系人信息；如用户未提供，则保留联系人、固定电话、手机占位符，用户明确要求删除时再去掉。
- 常用结尾：特此函复。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，宜写明回复事项。
- 主送单位放在标题下空一行顶格书写，末尾用全角冒号。
- 正文用 3 号仿宋体，首行缩进 2 字符；先回应来文依据，再逐项说明处理意见或答复结果。
- 如回复内容较多，可按问题序号分段；如答复简短，保持一函一事即可。
- 结尾可用“特此回复”“特此函复”，落款和日期置于文末右侧。

## 模板

```markdown
# 复函模板

## 标题

关于[事项名称]的复函

## 主送单位

[主送单位]：

## 来文回顾

[简要说明收到来函的时间、事项和背景。]

## 答复意见

[逐项答复对方关切，明确同意、不同意或需补充材料的内容。]

## 执行要求

[如需对方配合落实，可写后续对接方式、时间要求或责任分工。]

特此函复。

## 落款

[发文单位]  
[日期]

## 附注

（联系人：[联系人]，联系电话：[固定电话]，手机：[手机号]）
```
