# 命令（令）规范

## 写作规则

- 适用场景：公布规章、施行重大强制性措施、嘉奖有关单位和人员等。
- 结构重点：发布依据、命令内容、生效时间或执行要求、签署与日期。
- 写作提醒：仅在主体和权限匹配时使用，保持最高指令性。
- 分段职责：`发布依据` 写法定授权、批准程序或签发依据；`命令内容` 直接写公布什么、施行什么、采取什么措施或给予什么嘉奖；`生效说明` 写自何时起施行、适用范围和执行要求；`签署与日期` 集中写清签署人身份和时间，不分散表述。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：自公布之日起施行。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，可直接用“命令”或写明命令事项。
- 命令（令）类文本一般不设主送单位，可在标题下列命令号、签署人或生效说明。
- 正文用 3 号仿宋体，首行缩进 2 字符；命令事项宜表达得简短、明确、可执行。
- 签署人、签署日期、生效日期等关键信息应位置清楚，不要分散在多个段落中。
- 该类文本正式性强，自动导出只适合初稿，最终应严格套用法定模板。

## 模板

```markdown
# 命令（令）模板

## 标题

[发文主体]令  
第[文号]号

## 发布依据

[说明发布该命令（令）的法定依据或授权依据。]

## 命令内容

[写明公布、施行、嘉奖或采取措施的具体内容。]

## 生效说明

[写明施行日期、适用范围或执行要求。]

## 签署与日期

签署人：[姓名/职务]  
[日期]
```
