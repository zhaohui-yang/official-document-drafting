# 决议规范

## 写作规则

- 适用场景：会议讨论通过的重要决策事项。
- 结构重点：会议背景、审议通过事项、贯彻执行要求、落款。
- 要素分层：`必备` 为会议背景、决议事项、落实要求；`常见` 为会议届次或名称、决议事项分条、生效说明；`条件项` 为表决或通过情况、落款。
- 写作提醒：突出“经会议审议通过”的正式性，不写成一般性通知。
- 分段职责：`会议背景` 写会议届次、召开情况、讨论依据和形成决议的原因；`决议事项` 写经会议审议通过的事项、原则和结论；`落实要求` 写执行主体、时限和贯彻要求，突出“通过了什么、怎么落实”。
- 形成方式：决议应明确写出“经何种会议、在何时、以何种方式审议通过或表决通过”，这是决议区别于一般决定、意见、通知的关键识别要素。
- 文种边界：未经过法定会议、代表会议、全体会议或相应议事程序正式通过的事项，不宜写成决议；一般部署性、事务性安排应改用决定、意见、通知等文种。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：本决议自通过之日起施行。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，宜体现会议名称或决议主题。
- 决议一般不设主送单位，正文可按会议背景、表决结果、决议事项、落实要求展开。
- 正文用 3 号仿宋体，首行缩进 2 字符；会议通过事项宜逐条列示，确保结论醒目。
- 如涉及表决情况、通过日期、会议届次等信息，可放在标题后说明行或首段交代。
- 是否设置落款、署名和发布日期，应以会议或机关正式模板为准。

## 模板

```markdown
# 决议模板

## 标题

关于[事项名称]的决议

## 会议背景

[说明会议名称、召开时间、讨论审议背景和形成决议的依据。]

## 决议事项

一、[决议事项一]  
[写明会议表决通过或一致同意的事项。]

二、[决议事项二]  
[补充具体要求。]

## 落实要求

[明确执行主体、落实时限和工作要求。]

## 落款

[会议或发文主体]  
[日期]
```
