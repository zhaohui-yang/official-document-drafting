# 意见规范

## 写作规则

- 适用场景：对重要问题提出见解和处理办法。
- 结构重点：背景与总体要求、基本原则、主要意见、组织实施。
- 要素分层：`必备` 为背景与总体要求、主要意见、组织实施；`常见` 为基本原则、总体目标、重点举措；`条件项` 为责任分工、保障措施、附件。
- 写作提醒：重在原则和办法，不要写成单次事务通知。
- 分段职责：`背景与总体要求` 写现状问题、提出意见的必要性和总体目标；`基本原则` 写方向、底线和把握原则；`主要意见` 写制度安排、政策路径和具体办法；`组织实施` 写责任分工、推进机制、监督评估和保障措施。
- 常见起句：部委和省级样例中的意见，首段常先交代背景、依据和总体要求，再以“现提出如下意见”或相近句式总起全文。
- 文种边界：意见更适合中长期指导、制度安排和原则性处理办法；对一次性事务、短周期任务或只需简单布置的事项，不宜硬写成意见。
- 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注“待核实”或保留占位符。
- 常用结尾：请结合实际认真贯彻落实。

## 版式要求

- 基线要求：遵循共享总规则中的“版式与导出”。
- 标题通常用 2 号小标宋体居中排布，宜写成“关于[事项]的意见”。
- 意见类文本通常不单列主送单位；如确需明确对象，可在标题下另起一行标明。
- 正文用 3 号仿宋体，首行缩进 2 字符；背景、原则、主要意见、组织实施宜分别成层。
- 由于意见类文本常篇幅较长，一级标题和二级标题的字体层次要清楚，避免通篇大段堆砌。
- 如用于规范性政策传达，最终仍应与本单位正式政策文件模板和审签流程对齐。

## 模板

```markdown
# 意见模板

## 标题

关于[事项名称]的意见

## 背景与总体要求

[说明提出意见的背景、依据和总体目标。]

## 基本原则

[写明指导思想、基本原则或工作导向。]

## 主要意见

一、[意见一]  
[写明原则性安排、工作路径或处理办法。]

二、[意见二]  
[补充要求。]

## 组织实施

[明确责任分工、推进方式和保障措施。]

## 落款

[发文单位]  
[日期]
```
