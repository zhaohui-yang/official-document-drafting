# 行文规范

## 总体原则

- 保持正式、准确、简洁、可执行。
- 优先写明目的、依据、任务、要求，不堆砌空话。
- 不擅自添加法规依据、数字、文件号、会议结论。
- 用户未提供的关键信息，用占位符标示，不自行虚构。

## 标题写法

- 标题应直接点明事项和文种。
- 常见格式：`关于 + 事项 + 的通知`、`关于 + 事项 + 的请示`、`关于 + 事项 + 的报告`。
- 同一标题内避免同时出现多个中心事项。
- 标题一般居中排布；较长时可分行，但回行时要保持词意完整、排列对称。
- 标题中除法规、规章名称加书名号外，一般不用标点符号。

## 版式规范

- 详细版式说明见 `references/layout-rules.md`。
- 当前 skill 的 `.docx` 导出默认按 A4 页面和更接近机关公文的页边距处理。
- 默认标题按接近 2 号字处理，正文按接近 3 号字处理。
- 正文默认首行缩进 2 字符，采用固定行距，尽量接近每面 22 行、每行 28 字的常见排版密度。

## 正文结构

- 第一段优先交代背景、目的或依据。
- 第二层开始展开工作安排、请示事项或答复意见。
- 内容较多时使用层级编号：
- `一、`
- `（一）`
- `1.`
- `（1）`
- 一级标题建议统一使用 `一、二、三、四`
- 二级标题建议统一使用 `（一）（二）（三）`
- 三级标题建议统一使用 `1. 2. 3.`
- 四级标题建议统一使用 `（1）（2）（3）`
- 同一份文稿内不要混用多套编号体系，也不要跳级展开。
- 一般 10 页以内的文稿统一控制到二级标题，不再展开到三级及以下标题。
- 一般 2 至 3 页左右的文稿，使用一级和二级标题即可；通常不必继续下钻层级。

## 语言风格

- 用词稳健，避免口语、网络语、营销化表达。
- 多用动作导向表达，例如“开展”“落实”“报送”“组织实施”“严格执行”。
- 少用模糊词，例如“尽快”“适当”“有点”“比较好”。
- 需要强调要求时，优先写成可执行动作，而不是抽象口号。

## 层次标题写法

- 一级标题一般可写作：`一、XXXX`
- 二级标题一般可写作：`（一）XXXX`
- 三级标题一般可写作：`1. XXXX`
- 四级标题一般可写作：`（1）XXXX`
- 一级标题通常作为正文主体的大层次，承担并列事项、重点任务、主要问题、工作建议等一级展开。
- 二级标题通常用于对某个一级事项继续细分，适合部署类通知、综合性汇报、方案类材料。
- 10 页以内的文稿统一控制到二级标题，不再继续下钻。
- `一是、二是、三是` 通常用于段内分点衔接，不等同于正式二级标题；不要把 `一、` 和 `一是` 混作同层结构。
- 对“一是、二是、三是”等分点衔接语，默认应在同一自然段内连续书写，不得将每一点分别另起自然段。
- 除非用户明确要求逐条分段，或者单一点内容明显过长、确需单独强调，否则不拆分为多个自然段。
- 上述分点关键词宜与正文适度区分，常见做法是使用黑体或同等强调字体，后续说明内容仍保持正文字体。
- 一级标题一般不加句号。
- 二级标题以下如果是完整句式标题，可根据单位习惯决定是否使用句号；如无明确要求，优先保持简洁统一。

## 文种对应结尾

- 通知：`特此通知。`
- 请示：`妥否，请批示。`
- 报告：`特此报告。`
- 函：`特此函复。` 或 `特此函达。`
- 纪要：一般不写寒暄式结尾，直接收束到责任和要求。

## 落款与日期

- 落款通常包含发文单位和日期。
- 日期建议写成中文年月日形式，例如 `2026年3月15日`，并使用阿拉伯数字标明年月日。
- 年份应写全，月、日不补前导零，例如写 `3月5日`，不写 `03月05日`。
- 若用户未给出单位名称或日期，保留占位符，不要自行猜测真实主体。

## 主送机关与附件

- 主送机关通常置于标题下空一行，顶格书写，末尾用全角冒号。
- 有附件说明时，通常在正文下空一行左空二字编排“附件：”。
- 附件名称较多时，应按顺序号逐项列明。

## 质量检查清单

- 文种是否正确
- 行文方向是否合理
- 标题是否单一聚焦
- 标题是否存在不必要标点或回行不当
- 段落层级是否清楚
- 结尾用语是否匹配文种
- 落款和日期是否齐全
- 日期书写是否规范
- 是否出现未经提供的事实或依据
