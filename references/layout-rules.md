# 版式规范

本文件用于记录当前 skill 在正文主体排版上参考的公开规范口径，以及脚本已自动化和未自动化的部分。

## 适用边界

- 本 skill 当前重点处理“正文主体”的结构、字体、字号、缩进、行距和落款编排。
- 本 skill 当前可按 Markdown 可选章节生成“简化版头、发文字号、版记”和页脚页码。
- 本 skill 当前仍不生成完整机关套红版头、印章和公文用纸套红线。
- 若要形成可直接对外发出的正式机关红头文件，仍应在 Word 中按本单位模板做最后核定。

## 当前采用的版式口径

以下规则主要参照 `GB/T 9704-2012` 的公开解读材料和政府系统发布的操作说明整理：

- 用纸：A4
- 版心：156mm × 225mm
- 页边距：上 37mm、下 35mm、左 28mm、右 26mm
- 正文排版密度：一般每面 22 行、每行 28 字
- 标题：一般用 2 号小标宋体，居中排布
- 长标题：能一行则一行；确需回行时应词意完整、排列对称，优先逐行等长，不能完全等长时宜逐行递减，不应长短参差失衡
- 正文：一般用 3 号仿宋体；正文自然段通常首行缩进两格，文本成稿中可保留两个全角空格，导出时按首行缩进 2 字符处理
- 一级标题：一般用 3 号黑体
- 二级标题：一般用 3 号楷体
- 三级标题：一般用 3 号仿宋体加粗
- 四级标题：一般用 3 号仿宋体
- 正文中的实质性一级、二级板块标题，通常直接写作 `一、基本情况`、`二、主要做法`、`（一）重点任务` 等，并按“首行缩进 2 字符”处理
- 正文中的“一是、二是、三是”等分点关键词，默认与正文同段排布，不将每一点分别另起自然段；关键词可用黑体与正文区分
- 主送机关：标题下空一行，顶格，末尾用全角冒号
- 正文与落款之间：一般空 3 行
- 成文日期：用阿拉伯数字写明年月日，年写全，月日不补前导零，并按右空 4 字编排
- 附件：如有附件说明，通常在正文或结语下空一行左空 2 字编排“附件：”；多个附件时按顺序编号列示
- 图片类附件：图片、流程图、示意图、照片、截图等通常作为 `附件` 或 `附图` 单独编排，宜包含图号或附件号、标题和说明文字；技术指南类文本可使用 `附录`
- 附注：通常位于成文日期下一行，左空 2 字，加圆括号；请示类公文一般在附注处标联系人和联系电话
- 版记：标准公文版记常见要素通常为 `抄送机关、印发机关、印发日期`；`分送`、`主送移入版记`、`报/送/发` 等属于地方或系统样式；如设置版记，宜整体排在最后一页底部，当前页剩余空间不足时应整体移至下一页底部，不宜拆散

## 标题编排建议

- 标题应一行或多行居中排布。
- 能一行则一行；回行时应保持词意完整、排列对称，优先逐行等长，不能完全等长时宜逐行递减。
- 标题中除法规、规章名称加书名号外，一般不用标点符号。

## 当前脚本已自动处理的部分

- A4 页面尺寸
- 更接近规范的页边距
- 标题默认 22pt，正文与标题默认 16pt
- 标题、正文、一级/二级层次的默认字体区分
- 正文内“一是、二是、三是”等分点关键词可按黑体规则单独强调
- 版头默认使用单独字体槽位，可输出红色简化版头文字
- 发文字号区支持单行居中，或通过 `\t` 在同一行放置右侧“签发人”
- 版记区支持简化边线
- 页脚页码默认开启，首页不显示页脚页码，第二页起按自然页码显示；页脚本身不设置显式底纹或高亮；如需关闭，可使用 `--hide-page-number`
- 长标题默认按接近均衡的方式自动断行
- 正文固定行距由 `layout_profile` 控制；正式公文当前默认采用 `579 twips`（约 `28.95pt`），以更接近 225mm 版心下每面 22 行的推导值
- 正文首行缩进 2 字符；如 Markdown 或纯文本稿中已保留两个全角空格，导出时仍以段落缩进为准，不用半角空格堆砌版式
- 落款区分发文机关署名与日期的右侧编排，日期默认按右空 4 字输出
- 附注区支持按成文日期下一行输出，内容左空 2 字并用圆括号包裹

## 当前脚本未自动处理的部分

- 机关红头版头图片化或完整套红线
- 需要严格匹配标准位置的发文字号、签发人、抄送区和印发区
- 印章压成文日期
- 完整的附件页、附注区、抄送区、主题词区
- 长标题的人工审美修正，以及梯形/菱形等更细致的版面优化

## 使用建议

- 如果标题较长，可先让脚本自动断行；正式定稿前建议人工复核，必要时在 Markdown 中手动断行。
- 如需简化红头结构，可在 Markdown 中增加 `## 版头（可选）`、`## 发文字号（可选）`、`## 版记（可选）`。
- 如需在同一行中排“发文字号 + 签发人”，可在 Markdown 中使用制表符 `\t`。
- 如需关闭页码，在导出时增加 `--hide-page-number`。
- 如编辑器中页脚页码仍显示灰底，多数情况下是 Word/WPS 的字段阴影显示设置，不是文档本身写入了背景色或高亮。
- 如果文稿要正式发文，建议使用本单位现成的 Word 模板做最后套版。
- 如果只是形成“汇报稿、简报、专报、内部通知草稿”，当前脚本已能覆盖大部分正文主体排版需求。
